# 正则变量基础笔记

这份笔记是给读 Creamer、Zakharov 和自由面哈密顿理论时用的桥接材料。

目标不是堆定义，而是回答几个真正会卡住的问题：

- 正则变量到底是什么
- 为什么经典力学里常用 `(q,p)`
- 为什么水波里会用 `(\eta,\phi_s)`
- 泛函导数、Poisson bracket 到底在干什么
- 为什么大家会说这种形式“保留结构”或“尊重对称性”
- 为什么完整流体问题约化到自由面以后，`(\eta,\phi_s)` 还能继承 canonical bracket

## 1. 从最不抽象的图像开始

在普通经典力学里，一个质点常用两样东西描述：

- `q(t)`：位置
- `p(t)`：动量

它们不是随便选的两变量，而是因为在这组变量里，动力学能写成

```text
q_t =  dH/dp
p_t = -dH/dq
```

所以正则对的意思不是“能描述系统的一对变量”，而是：

**一对既能描述系统，又让运动方程具有标准哈密顿形式的变量。**

## 2. `(q,p)` 为什么特别

一个系统如果只求“描述状态”，很多变量都行：

- 位置和速度
- 能量和相位
- action-angle
- 位置和动量

但 `(q,p)` 特别，是因为它们让方程变成

```text
q_t =  dH/dp
p_t = -dH/dq
```

这个正负配对就是核心结构。

你可以把它理解成：

- `H` 告诉你“能量对各个方向怎么变化”
- 正则结构告诉你“如何把这个能量梯度转成真实运动”

所以系统不是像耗散系统那样沿着能量下降方向走，而是按一套特殊几何规则在相空间里流动。

## 3. 正则变量到底是什么

更本质地说，正则变量是在相空间里把辛结构写成标准形式的一组坐标。

一自由度时，标准辛形式是

```text
dp /\ dq
```

多自由度时是

```text
sum_i dp_i /\ dq_i
```

如果你现在还不想用微分形式语言，那就记实践版：

- 正则变量是使 Poisson bracket 具有标准形式的变量
- 也因此使 Hamilton 方程具有标准形式

这才是“正则”的真正含义。

## 4. 为什么这不是死记硬背

很多人第一次学正则变量时，会觉得像在背：

- 给你一对神奇变量
- 对 `H` 求导
- 方程就出来了

但更深的逻辑其实是：

一个保守系统的动力学不只由 `H` 决定，还由两部分共同决定：

1. `H` 本身
2. 把 `dH` 变成时间演化的几何规则

第二部分就是辛结构。

所以：

- `H` 告诉你系统的能量地形
- 正则结构告诉你系统如何在这个能量地形上运动

正则变量，就是让这套“如何运动”的规则写得最标准、最干净的一组坐标。

## 5. 为什么总是要两类变量

因为保守动力系统通常需要同时知道：

- 你“在哪里”
- 你“怎么穿过状态空间”

对质点来说：

- `q` 是位置
- `p` 是共轭动量

对场论来说也是一样，只不过自由度分布在空间上。

对水波来说，光知道自由面形状还不够。你还需要知道驱动它运动的流场信息。

这也是为什么自由面问题自然需要两类边界变量，而不是一个变量。

## 6. 从有限维走到场论

场论里，只是把有限多个 `(q_i,p_i)` 换成了空间上连续分布的场：

- `Q(x,t)`
- `P(x,t)`

对应方程变成

```text
Q_t =  delta H / delta P
P_t = -delta H / delta Q
```

差别只在于：

- 普通导数换成了泛函导数

因为现在 `H` 依赖的是整条函数，而不是几个数。

## 7. 水波里的 `(\eta,\phi_s)` 是什么

Zakharov 自由面理论里，常用的正则对是：

- `\eta(x,t)`：自由面高度
- `\phi_s(x,t)`：速度势在自由面上的取值

对应方程是

```text
eta_t   =  delta H / delta phi_s
phi_s,t = -delta H / delta eta
```

这就是场论版的

```text
q_t =  dH/dp
p_t = -dH/dq
```

所以：

- `\eta` 扮演“构型变量”的角色
- `\phi_s` 是它的共轭变量

## 8. 为什么偏偏是 `(\eta,\phi_s)`

这点最容易让人觉得像黑魔法，所以要说得很直白。

它们不是因为“看起来顺眼”才被选中，而是因为它们满足三件事：

1. 它们足以描述自由面势流问题的状态
2. 约化后的变分/哈密顿结构能在它们上面闭合
3. 方程在这组变量里正好成为标准哈密顿形式

物理上看：

- `\eta` 告诉你边界形状
- `\phi_s` 告诉你驱动这个边界运动的势函数边界值

这两者一起就构成了自由面势流问题的最小边界数据。

## 9. “能完整描述系统”还不够

你的直觉其实很接近正确答案，但还差一层。

确实，正则变量必须能完整描述系统。

但“完整描述”只是必要条件，不是充分条件。因为很多变量都能完整描述系统。

正则变量还必须额外满足：

- 它们之间的 Poisson bracket 是标准形式
- `H` 在它们上面直接生成运动

所以更准确地说：

**正则变量是一组特殊的完整变量，它不仅描述状态，还把动力学写成标准辛结构。**

## 10. 泛函导数到底在说什么

当你看到

```text
eta_t =  delta H / delta phi_s
phi_s,t = -delta H / delta eta
```

不要把它理解成“`eta` 和 `phi_s` 互相是导数”。

更好的理解是：

- `delta H / delta phi_s` 表示：如果你微调表面势，能量会怎么变
- `delta H / delta eta` 表示：如果你微调表面形状，能量会怎么变

然后正则结构把这些“能量对变量的敏感度”转成真正的时间演化。

所以这里的核心不是“谁是谁的导数”，而是：

**能量梯度通过正则结构生成运动。**

## 11. 为什么有个负号

一对正则变量满足

```text
q_t =  dH/dp
p_t = -dH/dq
```

这个负号很关键。

它保证系统的演化是 Hamiltonian 的，而不是耗散的。比如对自治系统，

```text
dH/dt = 0
```

也就是说，系统在流动时保能量、保相空间几何，而不是把能量耗掉。

## 12. Poisson bracket 到底在干什么

Poisson bracket 是把正则结构压缩成一个公式的方式。

一对 `(q,p)` 时：

```text
{F,G} = dF/dq dG/dp - dF/dp dG/dq
```

场论里对应的是

```text
{F,G} = Integral dx [
  delta F/deta   delta G/dphi_s
  - delta F/dphi_s delta G/deta
]
```

然后演化可以统一写成

```text
A_t = {A,H}
```

所以 Poisson bracket 的真正作用是：

**把“哈密顿量怎样生成运动”这件事写成一个统一算子规则。**

## 12.5 为什么 Poisson bracket 会带来阶次计数

这是后面理解 Creamer 第 3 节时特别重要的一点。

对一对正则变量 `(q,p)`，

```text
{F,G} = dF/dq dG/dp - dF/dp dG/dq
```

如果：

- `F` 是 `m` 次
- `G` 是 `n` 次

那么每一项都经历两步：

1. 先对 `F` 和 `G` 各求一次导
   - `dF/dq` 或 `dF/dp` 会把 `F` 的次数减 1
   - `dG/dq` 或 `dG/dp` 会把 `G` 的次数减 1
2. 再把两个导数结果相乘
   - 乘法会把次数相加

所以 bracket 的每一项次数都是

```text
(m-1) + (n-1) = m + n - 2
```

这就是为什么会有规则

```text
deg{F,G} = deg(F) + deg(G) - 2
```

这不是额外规定，而是 Poisson bracket 公式本身直接带出来的。

场论里的泛函 Poisson bracket 也是同样的逻辑：

- 泛函导数各把次数降一阶
- 然后两项相乘
- 所以总次数仍然按 `m+n-2` 计数

## 12.6 为什么这对 Creamer 特别重要

Creamer 第 3 节里最关键的三次消去条件是

```text
{H_2,W} = H_3
```

这时候阶数规则马上告诉你：

- `H_2` 是二次
- 如果 `W` 是三次
- 那么

```text
deg{H_2,W} = 2 + 3 - 2 = 3
```

刚好能和三次的 `H_3` 匹配。

同理：

```text
deg{H_3,W} = 3 + 3 - 2 = 4
```

所以 `H_3` 在 Lie transform 里的变化确实存在，但它从四次阶才开始影响当前计算，因此在“三次精度”下先不写。

这就是为什么在这一阶上只需要盯住

```text
H_3 - {H_2,W}
```

而不是整个无穷展开。

## 13. 为什么人们说这种形式“保留对称性”

这句话经常太抽象，所以最好翻译成人话。

当一个系统写成 Hamiltonian + Poisson 结构以后，连续对称性和守恒量之间的关系会变得非常自然。

例如：

- 时间平移对称性对应能量守恒
- 空间平移对称性对应动量守恒
- 旋转对称性对应角动量守恒

更进一步，很多守恒量本身就可以看成某种对称变换的生成元。

所以当人们说“这种形式保留了对称性”，通常是说：

- 系统的对称性和守恒量在这套语言里最清楚
- 正则变换不会破坏这套结构

## 14. 为什么正则变换重要

正则变换就是：

- 换变量
- 但不破坏哈密顿结构

这意味着：

- 你可以换到更好用的变量
- 但系统仍然是同一个 Hamiltonian 系统
- 运动方程仍然保有标准形式

这就是 Creamer 能做的事情：

- 不是发明了新物理
- 而是找了一组更好的哈密顿坐标

## 15. 为什么这对 Creamer 有用

原始变量 `(\eta,\phi_s)` 下，

- `H = H_2 + H_3 + H_4 + ...`

Creamer 做的是一个近恒等正则变换，使得：

- 非共振的 `H_3` 被搬出 Hamiltonian 主体

这样：

- 新变量仍然是正则变量
- 变换后的 Hamiltonian 更简单
- 但物理表面的非线性会在重建阶段重新出现

所以它的本质不是“换了一个海洋”，而是：

- 同一个系统
- 更好的坐标

## 16. 一个好记的心智图像

可以把正则变量理解成：

**适合描述运动的好坐标系。**

不是所有能描述状态的变量，都是适合做 Hamiltonian 理论的好坐标。

好的正则坐标满足：

- `H` 的变化能直接生成运动
- 守恒量和对称性容易看
- 进一步换变量时，可以保持这套结构

这样再回头看 Creamer 就容易很多：

- 它是在找更好的 Hamiltonian 坐标，而不是在发明另一套波动方程

## 17. 为什么 `(\eta,\phi_s)` 能继承 canonical bracket

这是最关键的桥接点之一。

短答案是：

**因为完整流体问题本身有变分/哈密顿结构，而在“不可压、无旋、自由面势流”这组约束下，体积内部的自由度可以被约化掉，最后留下的边界自由度正好就是 `(\eta,\phi_s)`，并且约化后的作用量仍然保持标准的 `p dq - H dt` 结构。**

我们一步一步说。

### 17.1 完整流体问题原本不是随便拼出来的

对无旋势流，可以用速度势 `\phi(x,z,t)` 描述流场。

自由面位置是 `z=\eta(x,t)`。

所以原始问题看起来是：

- 一个定义在流体域内部的场 `\phi`
- 一个 moving boundary `\eta`

这本来是个自由边界问题。

### 17.2 体积内部其实没有独立的“任意自由度”

因为对不可压无旋流，内部 `\phi` 必须满足 Laplace 方程：

```text
Delta phi = 0
```

这意味着内部场不是任意的，而是被边界数据决定的。

换句话说：

- 一旦给定自由面形状 `\eta`
- 再给定自由面上的势值 `\phi_s = \phi|_{z=\eta}`

内部的 `\phi` 就由调和延拓唯一决定

所以真正独立的数据，其实已经可以压缩到自由面上了。

### 17.3 约化后的作用量会变成边界形式

Luke 变分原理/Zakharov 约化的关键结果是：

在把内部 Laplace 约束用掉以后，作用量可以写成只含自由面变量的形式，核心结构是

```text
S = Integral dt [ Integral dx  phi_s eta_t  - H(eta,phi_s) ]
```

你可以把它和普通力学的

```text
S = Integral dt [ p q_t - H(q,p) ]
```

直接对照。

这已经几乎把答案写在脸上了：

- `phi_s` 扮演 `p`
- `eta` 扮演 `q`

因为作用量里出现的正是

```text
phi_s * eta_t
```

这样的正则一形式。

### 17.4 为什么这就意味着 canonical bracket

因为一旦作用量有

```text
Integral dx phi_s eta_t
```

这样的标准项，那么约化后的辛结构就是

```text
Integral dx dphi_s /\ deta
```

于是对应的 Poisson bracket 自然就是

```text
{F,G} = Integral dx [
  delta F/deta   delta G/dphi_s
  - delta F/dphi_s delta G/deta
]
```

这不是凭空指定的，而是从约化后的作用量直接读出来的。

所以“继承 canonical bracket”的真正意思是：

**完整问题的变分结构在约化到自由面后，没有消失，而是变成了自由面变量上的标准辛结构。**

### 17.5 为什么这不是巧合

这不是偶然的好看形式，而是“把内部受约束自由度消掉以后，边界变量成为真正的相空间坐标”的结果。

也就是说：

- 原系统有 Hamiltonian 结构
- 约束和约化做得对
- 剩下的自由度就会继承约化后的 Poisson/symplectic 结构

Zakharov 变量正是这个约化后的结果。

## 18. 这一节最该记住的句子

如果只记一句，就记这个：

**`(\eta,\phi_s)` 之所以是 canonical pair，不只是因为它们“够用”，而是因为在势流自由面问题里，内部自由度可由边界数据决定；把内部自由度约化掉后，作用量变成 `\int \phi_s \eta_t - H` 的标准形式，所以它们自然继承 canonical bracket。**

## 19. 现在最值得继续问的问题

当这份基础图像稳了以后，最自然的下一个问题是：

- 为什么消去 `H_3` 时，生成元 `W` 也必须取三次
- `H(\eta,\phi_s)` 里的 Dirichlet-Neumann operator 是怎么进来的
- 为什么它一展开就会产生 `H_2 + H_3 + H_4 + ...`
- 为什么非共振项可以被正则变换搬走
- 为什么 finite depth 里“最好的 canonical transform”和“最好的物理表面重建”可能开始分叉

## 20. 为什么消去 `H_3` 时，`W` 要取三次

这点非常值得记住，因为它把“Poisson bracket 的阶数计数”和
“为什么能消去 `H_3`”直接连起来了。

原始 Hamiltonian 写成

```text
H = H_2 + H_3 + H_4 + ...
```

我们希望通过 Lie transform 让变换后的 Hamiltonian 在三次阶上变成

```text
K = H_2 + O(4)
```

在三次精度下，关键修正是

```text
K = H_2 + H_3 - {H_2,W} + O(4)
```

所以为了消掉 `H_3`，我们需要

```text
{H_2,W} = H_3
```

现在做阶数计数：

- `H_2` 是二次
- `H_3` 是三次
- Poisson bracket 的次数规则是

```text
deg{F,G} = deg(F) + deg(G) - 2
```

因此如果要让

```text
{H_2,W}
```

正好也是三次，就必须有

```text
2 + deg(W) - 2 = 3
```

也就是

```text
deg(W) = 3
```

所以：

**`W` 取三次，不是随便选的，而是因为我们要用二次的 `H_2` 通过 Poisson bracket 生成一个三次对象，去抵消三次的 `H_3`。**

这也是为什么：

- `\{H_2,W\}` 是三次，可以拿来消 `H_3`
- 而 `\{H_3,W\}` 是四次，会被放到下一阶再处理

## 21. 生成函数到底是什么

在经典力学里，生成函数的作用不是“发明一个新系统”，而是：

**用一个标量函数，整体地定义一组新的正则变量。**

也就是说，它不是单独告诉你：

- `Q` 是什么函数
- `P` 是什么函数

而是给你一个更高一层的对象 `F`，然后通过对 `F` 求导，把新旧变量之间的关系一次性定义出来。

最熟悉的一种有限维写法是：

```text
F = F(q,P)
```

然后定义

```text
p =  dF/dq
Q =  dF/dP
```

这样得到的新变量 `(Q,P)` 会自动是正则变量。

所以生成函数的本质不是“多写了一个函数”，而是：

**它是一个把旧正则坐标变成新正则坐标的总说明书。**

## 22. 为什么生成函数能保证变换是正则的

最核心的原因是：

生成函数不是随便指定变换关系，而是通过作用量里的正则一形式来构造变换。

普通力学里，原来的正则项是

```text
p dq
```

如果变换以后你还能把它写成

```text
P dQ + dF
```

那么：

- 旧变量和新变量只差一个全微分 `dF`
- 全微分不会改变变分方程
- 所以辛结构保持不变
- 因而变换是 canonical 的

这就是为什么生成函数方法天然会保正则。

更直白一点说：

- 不是先随便写一个变量替换，再去检查它正不正则
- 而是一开始就从“保持正则结构”的要求出发来构造变换

## 23. 场论里为什么会变成“生成泛函”

对场论，变量不再是几个数，而是整条函数。

所以有限维里的生成函数会变成生成泛函。

在 Creamer 这篇 paper 里，写的是一个依赖旧变量和新变量的生成泛函：

```text
F(phi_s, zetaBar)
```

然后通过泛函导数定义变换：

```text
zeta(x)     = delta F / delta phi_s(x)
phiBar_s(x) = delta F / delta zetaBar(x)
```

这就是 paper `(3.2)` 和 `(3.3)` 的本质。

你可以把它和有限维的

```text
p = dF/dq
Q = dF/dP
```

直接类比。

所以 Creamer 里的 generating functional，本质上就是：

**场论版的正则生成函数。**

## 24. 为什么 paper 先写生成函数，再转 Lie transform

因为生成函数的好处是：

- 一开始就非常正统地说明“这是 canonical transform”
- 正则性是直接内建的

但它的缺点是：

- 变换关系通常是隐式的
- 真正做逐阶展开时不够灵活

所以 paper 先用 generating functional 说明：

- 我们做的确实是合法的正则变换

然后再切到 Lie transform，因为 Lie transform 更适合：

- 逐阶计算
- normal-form 消项
- 最后连接到 1D 的几何图像

所以：

- generating functional 更像“整体定义变换”
- Lie transform 更像“把这个变换拆成无穷小正则流”

它们不是两套不同目标，而是同一类正则变换的两种表达方式。

## 25. 关于生成函数，这一轮已经理顺的几点

下面这些点可以当成当前已经吃透的理解：

- 生成函数/生成泛函本身先给出的是“如何定义变换”的规则，而不是已经解好的显式变量替换。
- 所以它天然给出的是隐式关系：
  - 旧变量和新变量会混在同一组方程里
  - 真正要拿到具体 kernel 或具体变换公式，还要继续解这些条件
- 在 Creamer 里，这条路最后会落到 `(3.4)` 和 `(3.5)` 那些 kernel 条件上。
- 这些条件不是凭空出现的，而是三件事一起逼出来的：
  - 变换必须是 canonical
  - 变换后要消去 `H_3`
  - 生成泛函中的 kernels 必须满足相应的代数约束

关于 resonance，也已经有了一个很重要的分层理解：

- `delta^2(k_1+k_2+k_3)` 只负责波数闭合，也就是只决定哪些 triads 有资格出现在三次 Hamiltonian 里。
- 是否共振还要再看频率条件：

```text
+/- omega_1 +/- omega_2 +/- omega_3 = 0
```

- 所以：
  - `delta` 决定“允许耦合”
  - 频率分母决定“是否真正共振”
- 深水重力波里，这两件事不会同时把三波项变成真正 resonance：
  - triads 可以因为 `delta` 而出现
  - 但由于色散关系 `omega = sqrt(g |k|)`，对应分母不会为零
  - 所以 `H_3` 虽然存在，却仍然是非共振的

## 26. 关于 Lie transform，这一轮已经理顺的几点

下面这些点也可以当成当前已经理顺的主线：

- Lie transform 也不是“直接给出答案”，它同样要解方程，只不过解的是辅助参数 `lambda` 下的 canonical flow：

```text
partial_lambda A = {A,W}
```

- `lambda=0` 和 `lambda=1` 没有额外物理意义，只是：
  - `0` 表示还没变换
  - `1` 表示变换完成

- Lie series 看起来像 Taylor 展开，是因为它本质上就是沿 `lambda` 对 canonical flow 做展开。

- 为了消去三次项，需要把 `W` 当未知生成元去解

```text
{H_2,W} = H_3
```

而不是先拍脑袋指定 `W`。

- 频率分母的来源是：
  - `H_2` 负责线性振荡
  - 所以 `\{H_2,W\}` 作用在三模乘积上时，会拉出

```text
+/- omega_1 +/- omega_2 +/- omega_3
```

这样的频率组合
  - 当反过来解 `W` 时，这些频率组合就出现在分母里

- 因而：
  - `delta` 控制波数闭合
  - 频率组合控制 resonance
  - Lie transform 本身只是一个让这套逐阶消项逻辑变得透明的工具，不是额外神秘的新结构

## 27. 为什么 Lie flow 自动保 Poisson / canonical 结构

这一步是理解 Lie transform 时最后一个真正关键的概念点。

Lie flow 写成

```text
partial_lambda A = {A,W}
```

它和普通 Hamiltonian 时间演化

```text
A_t = {A,H}
```

在形式上完全一样，只不过：

- 真实时间 `t` 被换成了辅助参数 `lambda`
- Hamiltonian `H` 被换成了生成元 `W`

所以 Lie flow 本质上就是一种“以 `lambda` 为参数的 Hamiltonian flow”。

而 Hamiltonian flow 的关键性质就是：

- 保 Poisson 结构
- 因而也保 canonical 结构

为什么？

因为如果 `A(lambda)` 和 `B(lambda)` 都按

```text
partial_lambda A = {A,W}
partial_lambda B = {B,W}
```

演化，那么可以算出

```text
partial_lambda {A,B}
= {{A,W},B} + {A,{B,W}}
```

然后利用 Jacobi 恒等式

```text
{A,{B,C}} + {B,{C,A}} + {C,{A,B}} = 0
```

可以把它改写成

```text
partial_lambda {A,B} = {{A,B},W}
```

这说明 Poisson bracket 本身也在同一个 flow 下稳定演化，所以 bracket 结构不会坏掉。

而 canonical structure 本质上就是基本 Poisson bracket 的标准形式，例如

```text
{q_i,p_j} = delta_ij
{q_i,q_j} = 0
{p_i,p_j} = 0
```

既然 flow 保 bracket，这些关系也就会被保留下来。

所以更准确地说：

- 不是“Lie transform 先天神秘地正则”
- 而是“因为它被定义成 Hamiltonian / Poisson flow，所以它自动就是 canonical 的”

## 28. 关于 section 3，目前已经理顺的另一点

还有一个容易让人疑惑、但现在已经比较清楚的点是：

- 虽然 paper 介绍了 global generating function 和 Lie transform 两条路
- 但到了 Lie transform 阶段，作者仍然直接使用前面求出的 `B,D`

这不是偷懒，也不是自相矛盾。

原因是：

- 在“第一个非平凡阶”、也就是只做到三次精度并只关心消去 `H_3` 时
- 两种方法给出的变换关系是等价的
- 具体地说，只要把 Lie 生成元 `W` 取成 global generating functional 三次部分的负号
- 那么前面算出的 `B,D` 就可以直接拿来用

所以这一步最好的理解是：

- global generating function 先帮我们把 `B,D` 解出来
- Lie transform 再用这些 kernels 去组织更方便的 canonical flow

换句话说：

**在当前阶数上，`B,D` 是可以共用的；Lie transform 的真正优势，不在于重新发明一套不同的三次 kernels，而在于它对高阶结构和 1D 几何解释更自然。**

## 29. 关于 1D 简化，目前已经理顺的一点

进入 section 4 以后，一个关键理解是：

- 真正起作用的核心假设是 `1D`
- 再加上 triad 波数闭合

```text
k_1 + k_2 + k_3 = 0
```

而不是先额外假设 `k_1,k_2,k_3` 有某种固定大小顺序。

原因是：

- 在一维里，三个实数如果相加为零，就不可能全同号
- 必然有一个波数的符号和另外两个相反
- 因而自动推出：其中一个绝对值等于另外两个绝对值之和

这不是新的附加假设，而是 `1D + triad closure` 自动给出的几何事实。

正是这个事实让：

- `(3.4)` 的 `D` 分子必有一个因子为零，因此 `D=0`
- `(3.5)` 的 `B` 自动塌缩成只含 `|k|` 和 `sgn(k)` 的形式

所以 1D 的特殊性，首先来自 triad 闭合几何，而不是某个人为指定的大小排序。

## 30. 关于 section 4 的几何理解，目前已经理顺的一点

现在已经比较清楚的一条线是：

- section 3 先抽象地构造 canonical transform
- section 4 在 1D 里把它真正解成一个坐标重排问题

具体地说：

- `chi` 不是新物理量
- 它只是 characteristic 的标签，也就是“原来那个表面点的编号”
- 关系

```text
x = chi + lambda ZTilde_0(chi)
```

说明原来位于 `chi` 的点，被水平搬到欧拉位置 `x`

所以：

- `x` 是现在的位置
- `chi` 是原来的标签

这正好把抽象 Lie flow 重新翻译成了一个具体的坐标变换。

因此 `(4.13)` 最好理解成：

- 不是另一个新动力学方程
- 而是“已经解出的 1D 坐标重排”如何把线性变量重建成真实物理表面

## 31. 关于“重建”到底是什么意思

这里最容易混淆的地方是：

- 好像作者先有一套线性变量
- 然后又“重建”出一个新的表面

其实这里的“重建”不是重新求一个新的动力学解，而是在做：

**把参数坐标里的表面点集，重新翻译成欧拉坐标里的物理表面。**

要分清三件事：

- `chi` 或 `y`：点的标签，也就是参数坐标
- `x`：点最后出现的欧拉位置
- 场值：这个点本身携带的高度/势函数等信息

section 4 先通过 characteristic relation 解出：

```text
x = chi + lambda ZTilde_0(chi)
```

这句话的意思是：

- 编号为 `chi` 的点
- 被水平搬到了欧拉位置 `x`

而 `(4.13)` 做的，就是把这批已经被搬动的点，重新按固定空间位置 `x`
来读成物理表面 `zeta(x)`。

所以“重建”真正的意思是：

- 不是生成新的自由波
- 不是再求一遍动力学
- 而是把同一批点从参数表示翻译成欧拉表示

这也解释了为什么重建会自动带来：

- crest sharpening
- trough flattening
- higher harmonics

因为这些结构来自坐标翻译本身的非线性，而不是来自额外的自由传播模态。

## 32. 关于 section 4 里的 `y` 记号，一个容易混淆的点

在 1D 的 section 4 里，paper 里出现的 `y` 不应该理解成二维水平平面中的第二个方向坐标。

这里的 `y` 更接近：

- `chi` 那样的参数标签
- characteristic 的初始坐标
- 重建积分里用来标记“同一个表面点是谁”的变量

所以：

- `x` 是欧拉位置
- `y` / `chi` 是参数坐标

当 `(4.13)` 或 `(4.14)` 里写 `dy` 积分时，最好的心智模型是：

**我在对参数点集积分，再把它们翻译回欧拉表面；不是在沿几何上的另一个方向轴积分。**
